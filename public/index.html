POST /api/generate

Generate secure HEX code from personal information

Request Body:

{
  "firstName": "John",
  "lastName": "Smith",
  "countryCode": "US",
  "birthYear": "1990",
  "birthMonth": "12",
  "birthDay": "15",
  "gender": "male",
  "pin": 1234
}

Response:

{
  "status": "success",
  "hexCode": "A1B2C3D4E5F6789...",
  "debugInfo": {
    "originalSin": "110151408000000000000000000...",
    "securedSin": "123456789012345...",
    "components": {
      "firstName": "10151408000000000000000000",
      "lastName": "19130920080000000000000000",
      "country": "2119",
      "date": "19901215",
      "gender": "1"
    }
  }
}
POST /api/decode

Decode HEX code back to original personal information

Request Body:

{
  "hexCode": "A1B2C3D4E5F6789...",
  "pin": 1234
}

Response:

{
  "status": "success",
  "personalInfo": {
    "firstName": "John",
    "lastName": "Smith",
    "countryCode": "US",
    "birthYear": "1990",
    "birthMonth": "12",
    "birthDay": "15",
    "gender": "Male"
  }
}

Security & Implementation

Advanced security features and deployment guidelines

PIN-Based Encryption: The system uses mathematical transformation where the original SIN is multiplied by (PIN + 2025) and then has (PIN + 2025) added to it. This ensures unique HEX codes for identical personal information with different PINs.
Critical Security Notes:
  • PIN is never stored and cannot be recovered from HEX code
  • Incorrect PIN results in complete decoding failure
  • 2025 offset prevents mathematical vulnerabilities
  • Each component uses proprietary encoding algorithms
